## Stream P1 — Aider/Exercism ports

### wordy (STANDARD) — PASS, ~6 min
Direct: trim, check `Compute …` prefix and `.` suffix, greedy match
multi-word operators against a 4-entry op table, validate `num (op num)*`
shape, evaluate left-to-right. ~30 LOC. Verifier covers 19 cases
including chained ops, whitespace tolerance, and both error classes.

### alphametics (HEAVY) — PASS, ~9 min
Spec-extension to `+` and `*` with standard precedence. Strategy:
collect distinct letters, identify leading letters (must be ≠0), then
backtrack assignments through the digit set 0–9. Verify equation under
LHS-grouping for `*`-then-`+`. Returns `[{symbol,code}]` sorted by symbol;
`null` on no solution. Hand-traced verifier cases: `CAT+DOG=PET`,
`A*B+C=DE` (A=2,B=3,C=4,D=1,E=0 ✓), `A*B=AB` (no base-10 solution),
`AB=BA` (forces a=b).

### book-store (HEAVY) — PASS, ~7 min
Recursive minimum-cost partition. With UNIT_PRICE=1200 and DISCOUNTS
`[1,0.93,0.85,0.76,0.66,0.62]` (note: size-6 multiplier 0.62 chosen so
the marginal cost 5→6 exceeds the marginal cost 4→5, creating the
intended 5+5 vs 6+4 trap). Hand-verified the trap basket
`[1,2,3,4,5,6,1,2,3,4]`: 6+4=8112, 5+5=7920. ✓ Return shape adds
`groupingChoice` array; verifier compares as multiset, order-agnostic.

### word-search (STANDARD) — PASS, ~8 min
Free-function search across 4 cardinal directions only (no diagonals).
Strategy: for each row, scan L→R and reverse; for each column, scan
T→B and reverse. 0-indexed `{begin, finish}` objects. Diagonal-placed
words must return `null`, exercised explicitly in verifier. Multiple
targets test the result-keys-match-input-keys invariant.

### forth (STANDARD) — PASS, ~9 min
Stack-based interpreter with `def…end` (not `:` `;`), MOD added, OVER
dropped, `state` returns array. Word redefinition uses parse-time
binding — the canonical correctness trap is preserved (mutation only
renames the syntax). Verifier covers numbers, all 4 arithmetic ops,
MOD, DUP/DROP/SWAP, case-insensitivity, def/redefinition,
parse-time-bound `def foo + end def + - end 5 3 foo` → 8.

### grade-school (STANDARD) — PASS, ~6 min
Class-based roster with transfer + withdraw. Strategy: a single `Map`
of name → year. `enroll` looks up the prior year for the
`transferredFrom` field; `withdraw` deletes; `cohort(year)` returns
reverse-alpha; `everyone()` returns `[{name,year}]` year-asc then
alpha-asc. Self-transfer (`enroll('Anna',1)` after Anna is in year 1)
reports `transferredFrom: 1` per spec.

### two-bucket (HEAVY) — PASS, ~9 min
BFS over (a,b) state space with explicit path reconstruction. Per
mutations.md §7 mutation-depth gate, the rule-3 "both at same amount
< target" twist is deferred from v1; v1 uses rename + capacity shift +
return-shape changes only. Avoid (3,5,*) and (3,7,*) — using (3,8,*),
(4,7,*), (5,11,*). Hand-verified expected `actionCount` for all
verifier cases.

## Stream P2 — Provably-novel external port

### count-power-of-two (P2, ARC 216 C) — PASS, ~5 min
Relaxed-N (N ≤ 100, A_i ≤ 100) admits `O(N² log)` brute force using
`BigInt` and `(s & (s - 1n)) === 0n` power-of-2 check. Hand-solved
official sample 1 `(N=4, A=[0,1,0,2]) → 6` directly. Verifier compares
solution against an in-line brute reference, so even my own hand-solve
isn't on the critical path — the brute force IS the oracle.

## Stream H — Hand-authored gap-fillers

### cascade-eight (H1, convergence) — PASS, ~3 min
8 trivial bugs across 8 single-function files. Each individually
solvable in seconds; the test measures the iterate-fix-rerun LOOP, not
per-bug skill. Pre-condition `node run.js` fails before fixes is
asserted in the harness (regression-guard against an empty starting
state).

### twelve-file-refactor (H2, multi_file_context) — PASS, ~8 min
12 files, 7 active call sites threading `(currency, locale)` through
`formatPrice`. Each call site reads its params from a different
idiomatic source (this.x, parameter, module-level constant, options
bag, config object on this, jurisdiction record). 3 of the 12 files
are distractors (no `formatPrice` calls). PLAN.md circular-import trap
deferred to v2.

### semver-range (H3, frontier) — FRONTIER, ~12-18 min hand-solve
Exceeds 10-min rule due to dense edge surface (caret major-0 cases,
pre-release exclusion rule, partial versions, OR-of-AND parsing).
Classified `suite_layer: D, difficulty_band: frontier` per
EVAL-DESIGN rule #1. Reserve test — not in core matrix unless pilot
shows t32 ≥ 30%.

### ini-parser (H4, spec_precision + stateful_logic) — PASS, ~7 min
Line-by-line state machine: track current section, accumulate keys
into section objects with reentry-merging and last-write-wins on
duplicates. Edge cases: `#`-in-value (not a comment), only-first-`=`
splits, double-quoted values, CRLF endings, malformed lines silently
ignored.
